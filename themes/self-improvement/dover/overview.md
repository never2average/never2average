# Dover

Security that defends itself. Named for Dover Castle, the most heavily fortified castle in England.

The world ships more code every day, and most of it is composed rather than written, so supply-chain risk compounds with every release. Today's tools cannot keep up: they enforce static rule libraries, bury teams in logs, or wait for a CVE to be published long after the door was already open. Dover is a generative security harness that thinks like an attacker and acts like a teammate. It sandbox-clones your entire application, finds the vulnerabilities that are genuinely exploitable on the clone, and opens pull requests on the real thing, so your security posture improves on its own, at the speed you ship.
