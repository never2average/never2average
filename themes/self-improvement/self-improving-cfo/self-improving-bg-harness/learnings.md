# Learnings

The first learning is that recommendations are not enough. A finance operator that only alerts me is still making me do the function. The background harness has to take spend-linked actions, not just point at problems.

The second learning is that the macro-to-micro translation is the hard part. A vague strategy is not enough. The macro goal has to be quantified, and the harness has to convert it into small spend decisions without losing the original strategy.

The third learning is that the full-stack app is a necessary bridge. Before the background harness can act, users need to be set up, permissions need to be visible, and enough sessions need to exist for the system to learn what the operator actually wants. It needs cash flow, token usage, model usage, infra bills, revenue, runway, pricing experiments, and prior sessions.

The background version is only useful if it changes the shape of the work. One approval workflow, one accurate spend policy, and one recurring forecast would already be a real first version.
