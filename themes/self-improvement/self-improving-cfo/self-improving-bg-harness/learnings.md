# Learnings

The first learning is that recommendations are not enough. A finance operator that only alerts me is still making me do the function. The background harness has to create and route tasks, not just point at problems.

The second learning is that the macro-to-micro translation is the hard part. A founder can say the company should protect runway, cut inference cost, or push harder on growth. The harness has to convert that into small spend decisions without losing the original strategy.

The third learning is that the full-stack app is a necessary bridge. Before the background harness can act, users need to be set up, permissions need to be visible, and enough sessions need to exist for the system to learn what the operator actually wants.

The background version is only useful if it changes the shape of the work. It should make finance feel less like an app I use and more like a function that runs.
