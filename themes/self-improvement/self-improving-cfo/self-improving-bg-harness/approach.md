# Approach

The self-improving background harness comes from a simple feeling: the finance function should disappear into the background. Even the full-stack Thin Wedge app, used locally, can feel like a faster horse. It is better than the CLI for permissions, forecasts, and navigation, but it is still something I have to operate.

A car-like experience is different. I set the direction and the financial goals for the business. The harness has enough finance agents and CLI access to start managing spend around that direction.

That means actual task making, not just alerts. It should project spend, approve or deny spend at runtime, edit spend policy, and translate broad financial strategy into micro decisions. Some actions might be approved. Some rejected. Some constrained by usage limits or other policy limits.

The point is not to ask the CFO agent every question. The point is to give it the agreed financial perspective and let it carry that into day-to-day execution.
