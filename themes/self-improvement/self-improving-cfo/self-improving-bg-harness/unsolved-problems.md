# Unsolved Problems

The biggest blocker is sequencing. The full-stack Thin Wedge implementation has to be finished first. People have to get set up on it. There need to be enough real sessions and user data before it makes sense to migrate them into an action-taking framework.

The second blocker is policy execution. The system has to take a macro financial strategy and turn it into micro approvals, denials, rejections, usage limits, and spend constraints. That is much harder than producing a forecast.

The open questions are concrete:

- Can the harness approve and deny spend at runtime without violating the founder's financial intent?
- Can it edit spend policy without becoming dangerous?
- Can it create the right tasks instead of only creating notifications?
- Can it preserve enough audit trail that every micro decision can be traced back to the macro strategy?

The background harness works when I can set the company's broad financial direction and trust the system to carry it into the small decisions I would otherwise have to keep making myself.
