# Unsolved Problems

The biggest blocker is sequencing. Thin Wedge Full Stack has to be finished first. People have to get set up on it. There need to be enough real sessions and user data before it makes sense to migrate them into an action-taking framework.

The second blocker is policy execution. The system has to take a macro financial strategy and turn it into micro approvals, denials, rejections, usage limits, and spend constraints. That is much harder than producing a forecast.

The open questions are concrete:

- Can the harness approve and deny spend at runtime without violating the founder's financial intent?
- Can it edit spend policy without becoming dangerous?
- Can it cap workflows and approve token-heavy experiments for the right economic reason?
- Can it preserve enough audit trail that every micro decision can be traced back to the macro strategy?
- Can it make sure only an authorized person can approve spend?
- Can it survive prompt injection when money-moving permissions are nearby?

The scariest failure mode is bad translation: a correct macro-level financial constraint becoming an inaccurate micro-financial constraint. Thin Wedge Auto works when I can set the company's broad financial direction and trust the system to carry it into the small decisions I would otherwise have to keep making myself.
