# Learnings

The first learning is that the surface matters because trust is the product. A local CLI is not glamorous, but it is believable. It runs where the user works. It does not ask them to move sensitive financial context into a new SaaS product before the tool has earned the right.

The second learning is that token spend is not an engineering footnote. In an AI-native company it can be the main cost center. It needs budgets, attribution, limits, forecasts, and business-goal tradeoffs. That is not how normal controllers think yet.

The third learning is that AI finance work needs structured data and sandboxes much earlier than expected. The hard part was not only making an agent talk about finance. It was getting enough structure and safe execution around the workflows so the answers could be trusted.

Thin Wedge exists because the finance person for a solo AI product operator cannot just be a spreadsheet person. It has to understand tokens, models, infra, pricing, experiments, and product strategy in the same loop.
