# Unsolved Problems

The real blockers are the same reasons this version exists.

The first blocker is security permissions. The app has to make it clear what the agent can see, what it can change, and what still needs approval. Without that, a finance web app is too risky to trust.

The second blocker is visualization. The product has to show token spend, forecasts, attribution, and pricing experiments in a way that makes the financial state easier to understand than a CLI transcript.

The third blocker is scheduled tasks. The app needs to run finance workflows on a cadence without turning into a noisy notification machine.

The vertical AI web app works when those three things feel natural: permissions are obvious, the financial picture is visible, and scheduled tasks keep the operator ahead of spend instead of reacting late.
