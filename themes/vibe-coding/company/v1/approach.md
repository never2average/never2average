# Approach

Relay is the collection-of-agents version of vibe coding a company. The founder expresses an objective in language, and the system decomposes it into agent roles, workflows, and review loops that can move the company forward without forcing the founder through a dashboard of forms.

The Relay approach is intentionally direct:

- Start with the founder's raw objective.
- Create a small set of agents with clear responsibilities.
- Let each agent produce work artifacts, not just chat.
- Use human review to decide what becomes durable company state.

Relay is less about building the final operating system and more about proving the primitive: can language pass intent between agents, produce useful work, and move a company workflow faster than a founder could wire it manually?
