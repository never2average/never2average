# Learnings

The first learning is that "vibe coding a company" cannot mean "ask an agent to do everything." That becomes noise very quickly. Relay has to guide the founder through the journey. It has to tell them which kind of agent helps at which stage, what evidence matters, and what should be done next.

The second learning is that the sequence matters. In the CFO-suite journey, the CLI came first because trust, local setup, and speed mattered more than a polished SaaS surface. Then permissions, visualizations, scheduled forecasts, and API integration forced the full-stack version. Then the background CFO harness only made sense after there were enough real sessions and enough data. That ordering was not cosmetic. It was the company-building path.

The third learning is that agent handoffs are the product. A founder should not have to re-explain the company every time the work moves from customer discovery to agent boundaries, from agent boundaries to evals, or from evals to FinOps and pricing. Relay means relaying context between agents well enough that each stage compounds.

The fourth learning is that every step has to become measurable. A 26-week playbook is only useful if Week 6, Week 10, Week 16, Week 20, and Week 26 have real outputs. Otherwise it is just a motivational roadmap with agents attached.
