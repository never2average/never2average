# Approach

Cabinet is the operating-system-style version of vibe coding a company. Relay proves that a founder's objective can create useful agents and workflows. Cabinet turns that into a persistent company surface where goals, agents, decisions, artifacts, metrics, and operating memory live together.

The product should behave less like a dashboard and more like a company runtime:

- Objectives become durable work streams.
- Agents have roles, context, permissions, and review responsibilities.
- Workflows produce artifacts that can be inspected, reused, and improved.
- Company state changes only through traceable decisions.

The interface should still start with language. The structure should appear because the company needs it, not because the user had to design a process before asking for work.
